# 🚀 后端配置教程 · 5 分钟让排行榜全网同步

本项目后端用 **Supabase**（免费云数据库）。代码已全部写好，你只需注册账号 + 建表 + 填凭据，**改完立即全网共享排行榜**。

---

## 为什么用 Supabase

- ✅ 免费额度（500MB 数据 + 无限请求）对娱乐网页绰绰有余
- ✅ 全球 CDN，访问快
- ✅ 自带匿名写入，不用搞用户注册
- ✅ 数据库在 Supabase 那边，你的电脑/服务器都不用动，不怕被攻击
- ✅ 项目代码**第一期就写好了**，填完凭据零代码改动

---

## 第 1 步：注册 Supabase（2 分钟）

1. 打开 **https://supabase.com**
2. 右上角点 **`Start your project`** → 用 GitHub 账号登录（最快）
3. 同意授权后会进入控制台
4. 点 **`New project`**（新建项目）
5. 填写：
   - **Name**：`gaokao-time-machine`（随便起）
   - **Database Password**：点 `Generate a password` 自动生成，**复制保存好**（丢了要重置）
   - **Region**：选 `Southeast Asia (Singapore)` 或 `Northeast Asia (Tokyo)`（离中国近）
   - **Pricing Plan**：选 **`Free`**（免费版）
6. 点 **`Create new project`**，等 1-2 分钟初始化

---

## 第 2 步：建表（1 分钟）

1. 项目创建好后，左边菜单点 **`SQL Editor`**（有个 `>` 图标）
2. 点 **`New query`**
3. 把下面**整段 SQL** 复制粘贴进去：

```sql
create table if not exists public.box_records (
  id            bigint generated by default as identity primary key,
  label_id      text        not null,
  label_name    text        not null,
  tier          text        not null,
  province      text        not null,
  province_name text        not null,
  exam_year     int         not null,
  score         int         not null,
  rank          int,
  track         text,
  track_label   text,
  major         text,
  equivalent_score_2026 int,
  created_at    timestamptz not null default now()
);

create index if not exists idx_box_records_created_at
  on public.box_records (created_at desc);
create index if not exists idx_box_records_province
  on public.box_records (province);
create index if not exists idx_box_records_label
  on public.box_records (label_id);

alter table public.box_records enable row level security;

drop policy if exists "anon insert box_records" on public.box_records;
create policy "anon insert box_records"
  on public.box_records for insert to anon with check (true);

drop policy if exists "anon select box_records" on public.box_records;
create policy "anon select box_records"
  on public.box_records for select to anon using (true);
```

4. 点右下角 **`Run`**（运行），下方出现 `Success` 就成了

---

## 第 3 步：拿到凭据（30 秒）

1. 左边菜单点 **`Project Settings`**（齿轮图标）→ **`API`**
2. 找到这两项，复制下来：
   - **`Project URL`**：形如 `https://abcdefgh.supabase.co`
   - **`anon public` key**（在 API Keys 里，**不是** service_role）：一长串 `eyJhbGci...` 开头

> ⚠️ `anon` key 是公开的（设计如此，前端要用），配合 RLS 策略只能做受控的增删改查，安全。**千万别用 service_role key**，那个有完整权限。

## 第 4 步：填进项目（30 秒）

打开文件 **`js/config/supabase.js`**，找到第 18-19 行：

```js
  const SUPABASE_URL = '';        // 例：'https://xxxxx.supabase.co'
  const SUPABASE_ANON_KEY = '';   // 例：'eyJhbGciOi...'
```

填上你刚复制的两个值：

```js
  const SUPABASE_URL = 'https://abcdefgh.supabase.co';     // 你的 Project URL
  const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR...';  // 你的 anon key
```

保存。

## 常见问题

**Q: 控制台报错 "supabase is not defined"？**
A: CDN 加载失败（网络问题），刷新重试。Supabase JS 通过 jsdelivr CDN 加载，国内偶发慢。

**Q: 报错 "permission denied" / "RLS"？**
A: 第 2 步的 RLS 策略 SQL 没跑或跑错了，回 SQL Editor 重跑一遍那两段 `create policy`。

**Q: 数据库里能看到记录但前端排行榜是空的？**
A: 检查 `anon` key 是否填对（不是 service_role key）。

**Q: 想重置排行榜数据？**
A: Supabase 控制台 → Table Editor → `box_records` → 全选删除，或跑 `TRUNCATE public.box_records;`

**Q: 免费额度会用完吗？**
A: 500MB 大概能存 50 万条记录，娱乐网页很难用完。请求无限。

## 安全说明

- 你填的是 **anon key**（公开 key），配合 RLS 策略只能：匿名**插入**新记录、匿名**查询**所有记录。**不能删除/修改**别人数据（RLS 没开放 update/delete）。
- 这正是为娱乐网页设计的：大家都能写入、都能看排行榜，但谁也改不了谁的数据。
- service_role key 千万别放前端，那个能删库。
